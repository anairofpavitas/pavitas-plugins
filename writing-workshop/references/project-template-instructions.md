## How We Work

This project uses the **writing-workshop** plugin (writing-interview + style-mirror).

**The typical session:**
1. Mount the project folder: `[path to project folder]`
2. Say what scene or section we're working on
3. The writing-interview skill kicks in — one question at a time, pulling the scene out through conversation
4. When I say compile, it assembles a rough draft using the style profile (the `-style.md` file in the project folder)
5. Draft gets saved to the project folder with an appendix of suggestions, unanswered questions, and logical deductions

**Don't do these things:**
- Don't suggest plot directions unless I ask. Extract, don't co-plot.
- Don't name emotions in the prose. Show through the body.
- Don't inject content from the style profile's examples into new scenes. The profile captures technique, not story material.
- Don't polish the draft. It should read rough — connected prose, but not refined. I'll do that part.
- Don't pad the interview with unnecessary setup questions. If I've given you context, skip ahead to the first real question.

**Do these things:**
- Follow the thread of whatever I'm most energized about. If I give a long answer, ask one more question to go deeper, then move on.
- If I say "I don't know yet" or "I'll figure that out later," respect it. Log it in the appendix and move on immediately.
- If I start flowing and giving you polished prose, get out of the way. Just ask "what happens next?" when I pause.
- Use my words in the compiled draft. If I said it a certain way, keep it that way.
