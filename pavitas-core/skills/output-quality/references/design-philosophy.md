# Design Philosophy

## Core Conviction

**Generic is a choice, not a constraint.** Every time Claude delivers a visual output that looks like a first draft, it's because design thinking wasn't applied, not because better wasn't possible.

## The Design Director Mindset

Think like someone who wouldn't let mediocre work leave the studio:

1. **Question every default** - PowerPoint's default blue? Google Sheets' standard gray? These are starting points, not destinations.

2. **See through technique to intent** - Don't just add white space; understand what breathing room accomplishes. Don't just choose a typeface; know what it communicates.

3. **Balance boldness with restraint** - Professional design isn't about using every technique at once. It's about selecting the right move for the right moment.

4. **Craft, don't template** - The goal is outputs that look hand-crafted by someone who cares, not outputs that look generated.

## Balancing Expertise with Restraint

### When to Push Bold

- **Hierarchy needs clarity** - Use dramatic scale differences, strong color contrast, or distinctive typography when establishing clear information hierarchy
- **Brand demands presence** - Bold accent colors, confident spacing, distinctive voice when representing a brand or personality
- **Complexity needs taming** - Strong structural moves when organizing dense information (dashboards, reports, data visualization)

### When to Exercise Restraint

- **Content is king** - Let compelling content breathe; don't compete with it
- **Context is conservative** - Corporate presentations, financial reports, academic contexts often benefit from understated sophistication
- **Simplicity is the goal** - Sometimes the most powerful design move is removing everything unnecessary

### The Sophistication Spectrum

**Minimal ≠ Generic**
- Generic: Default fonts, standard spacing, predictable layouts
- Minimal: Deliberate typography, intentional white space, refined proportions

**Bold ≠ Loud**
- Loud: Using every technique simultaneously, competing visual elements, confusion
- Bold: One strong choice that defines the piece, supporting elements that reinforce

**Professional ≠ Safe**
- Safe: Avoiding any distinctive choice, hiding behind defaults
- Professional: Confident choices executed with precision and purpose

## The Elevation Standard

Every visual output should answer "yes" to these questions:

1. **Would this hold up in a portfolio?** Not as a placeholder, but as a finished piece you'd show a prospective client.

2. **Does every choice have intent?** Can you explain why this typeface, this color, this spacing, this layout?

3. **Is it distinctively crafted?** Could someone tell this wasn't generated from a generic template?

4. **Does it serve the content?** Does the design enhance understanding and impact, or just decorate?

## Working Philosophy

### Start Functional, Then Elevate

1. Build a working version that accomplishes the user's goal
2. Apply design interrogation (see interrogation-checklist.md)
3. Reference professional standards (see reference-library.md)
4. Deploy specific techniques (see technique-catalog.md)
5. Follow systematic refinement (see elevation-protocol.md)

### Think Systems, Not Decoration

Design isn't applied afterward like frosting. It's baked into every choice:
- Typography is how hierarchy communicates
- Color is how attention directs
- Space is how breathing room creates
- Layout is how relationships form

### Embrace Constraints as Creative Fuel

Working within presentation software? HTML limitations? Spreadsheet grids? These aren't obstacles to design thinking—they're parameters that force creative solutions. The best design often emerges from solving real constraints elegantly.

## The Invisible Work

The user should see:
- A polished, professional output that looks hand-crafted
- Clear hierarchy and easy navigation
- Distinctive style that feels intentional
- Content that's enhanced by its presentation

The user should NOT see:
- Generic templates or obvious defaults
- Design that calls attention to itself
- Unnecessary complexity or decoration
- Any sense that this is a first draft

## When in Doubt

**Ask: What would the design director version of this choice look like?**

Then make that choice.
