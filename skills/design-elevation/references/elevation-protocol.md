# Elevation Protocol

A systematic process for taking functional outputs to polished, professional quality. This happens internally—the user sees only the elevated result.

## The Protocol: Functional → Elevated

### Phase 1: Build Functional Foundation (Quick)

**Goal:** Get the content working and structured

**Actions:**
1. Create basic structure that accomplishes the user's goal
2. Ensure all requested information is present
3. Verify functionality (formulas work, links function, data displays)
4. Establish basic hierarchy (headers, sections, body text)

**Output at this phase:** 
- Functional but generic
- Default styling
- Content is there but unremarkable
- This is what most outputs stop at—don't let yours

**Time investment:** 20-30% of total effort

### Phase 2: Design Interrogation (Critical Thinking)

**Goal:** Question every default choice

**Process:**
1. **Read the interrogation checklist** (`references/interrogation-checklist.md`)
2. Work through each category relevant to this output type
3. Identify what's currently generic or default
4. Make specific notes about what needs elevation

**Key questions to ask yourself:**
- What am I using because it's default vs. because it's right?
- Where is hierarchy unclear or weak?
- What's competing for attention when it shouldn't be?
- Where could dramatic choices create impact?
- What would make this feel hand-crafted, not generated?

**Output at this phase:**
- Clear understanding of design problems
- Specific plan for elevation
- Reference materials identified (which exemplars to study)

**Time investment:** 15-20% of total effort

### Phase 3: Reference & Technique Selection (Strategic)

**Goal:** Choose the right moves for this context

**Process:**
1. **Identify primary job:** Hierarchy? Professionalism? Attention? Information density?
2. **Reference appropriate exemplars** (`references/reference-library.md`)
   - Building a dashboard? → Study Linear, Figma
   - Creating a presentation? → Study Apple, Stripe
   - Making a document? → Study Swiss Design, Notion
3. **Select techniques** (`references/technique-catalog.md`)
   - Pick ONE primary technique (e.g., dramatic scale differences)
   - Add 1-2 supporting techniques (e.g., precision spacing, selective color)
4. **Consider design philosophy** (`references/design-philosophy.md`)
   - Does this context call for bold or restraint?
   - Where should I push, where should I hold back?

**Output at this phase:**
- Specific design direction
- 2-3 concrete techniques to apply
- Understanding of appropriate style for context

**Time investment:** 10-15% of total effort

### Phase 4: Systematic Refinement (Execution)

**Goal:** Apply chosen techniques consistently throughout

**Order of operations:**

#### 4a. Typography Elevation
**Apply first** (typography is foundation of hierarchy)

1. Replace default fonts with intentional choices
   - Presentation/dashboard: Inter, SF Pro, Space Grotesk
   - Document: Inter + Crimson, or Lora
   - Technical: Inter + JetBrains Mono

2. Establish dramatic scale differences
   - Headers 3-4x body size minimum
   - Labels 0.75x body size
   - Create clear visual levels

3. Set proper line heights
   - Display text: 1.2-1.3
   - Body text: 1.5
   - Don't use single spacing

4. Apply weight hierarchy
   - Headers: 600-700
   - Body: 400
   - Labels: 300-400

**Check:** Can you identify information priority in 2 seconds?

#### 4b. Color Elevation
**Apply second** (color reinforces hierarchy)

1. Establish color system
   - Choose 1 primary, 1 accent, gray scale
   - Reference appropriate palette from reference-library.md
   - Test contrast ratios (4.5:1 minimum for body text)

2. Apply color functionally
   - Primary: Brand, key actions
   - Accent: Emphasis, interactive states
   - Grays: Structure, subordinate information
   - Semantic: Green (success), Red (error)

3. Use color restraint
   - Mostly grayscale (70-80%)
   - Accent color in <10% of layout
   - Don't colorize everything

**Check:** Is color directing attention purposefully?

#### 4c. Layout Elevation
**Apply third** (layout organizes everything else)

1. Establish spacing system
   - Choose 8pt or 12pt scale
   - Apply consistently: 8/16/24/32/48/64/96
   - Increase margins generously (48-96px minimum)

2. Create grid structure
   - Align everything to invisible grid
   - Use column layouts (2, 3, or 12 column)
   - Break grid only intentionally

3. Apply grouping principles
   - Related items closer together (16-24px)
   - Unrelated items further apart (48-64px)
   - Use white space to create relationships

4. Position key elements intentionally
   - Don't center everything
   - Use asymmetry for interest
   - Create focal points through placement

**Check:** Does the layout guide the eye naturally?

#### 4d. Format-Specific Polish
**Apply last** (medium-specific refinement)

**For Presentations (PPTX, Google Slides):**
- [ ] One key point per slide (rest in notes)
- [ ] Slide master customized (not default template)
- [ ] Images full-bleed or dramatically sized
- [ ] Consistent header/footer positioning
- [ ] Build animations for progressive reveal

**For Spreadsheets (XLSX, Google Sheets):**
- [ ] Custom header styling (not default green/blue)
- [ ] Subtle row banding (gray-50 alternating)
- [ ] Number formatting (currency, decimals)
- [ ] Borders only where structure demands
- [ ] Conditional formatting (if helpful)
- [ ] Gridlines hidden

**For HTML/Web Artifacts:**
- [ ] CSS Grid or Flexbox for layout
- [ ] Custom CSS properties for theming
- [ ] Hover states on interactive elements
- [ ] Smooth transitions (0.2s ease)
- [ ] Responsive spacing (clamp, viewport units)

**For Documents (PDF, DOCX):**
- [ ] Generous margins (1-1.5 inch)
- [ ] Running headers/footers
- [ ] Page numbers consistent
- [ ] Section breaks clear
- [ ] High contrast for readability

**Check:** Does this feel polished for its specific medium?

### Phase 5: Final Quality Gate (Verification)

**Goal:** Ensure nothing generic remains

**Final review process:**

1. **Compare against functional version**
   - What changed? Is it dramatically better?
   - If differences are subtle, elevation didn't happen enough

2. **Run through interrogation checklist again**
   - Typography: Dramatic scale? Intentional choices?
   - Color: Functional use? Restrained palette?
   - Layout: Clear hierarchy? Generous spacing?
   - Format-specific: Medium-appropriate polish?

3. **Ask the critical question:**
   **"Would I show this to a design director without apologizing?"**
   - If yes → Ship it with confidence
   - If no → Identify what's still generic and fix it
   - If "it's good enough" → Remember that's the enemy of distinctive

4. **Check for consistency**
   - Same fonts throughout
   - Same spacing system throughout
   - Same color usage patterns throughout
   - Alignment consistent

5. **Remove any hedging or apologizing in presentation**
   - Don't say "Here's a draft" or "This is a start"
   - Don't preemptively apologize for design choices
   - Present with confidence—the design work happened internally

**Time investment:** 35-45% of total effort

## Accelerated Protocol (For Simple Outputs)

When creating simpler outputs (single slide, basic table, short document), use compressed version:

### Fast Track (3 Steps)

1. **Build + Quick Check** (30%)
   - Create functional version
   - Scan interrogation checklist for obvious problems
   
2. **Apply Top 3 Moves** (50%)
   - Dramatic scale for hierarchy
   - One strong color choice
   - Generous spacing
   
3. **Polish Pass** (20%)
   - Format-specific refinement
   - Remove defaults
   - Verify consistency

## Common Elevation Patterns

### Pattern: Dashboard Creation

1. Functional: Basic data display, default styling
2. Interrogation: Hierarchy weak, colors random, spacing tight
3. Reference: Linear (dark mode, opacity hierarchy), Figma (information density)
4. Techniques: Card-based layout, dramatic number sizing, selective color
5. Polish: Rounded corners, subtle shadows, consistent spacing
6. Result: Professional dashboard that guides attention

### Pattern: Presentation Creation

1. Functional: Content on slides, default template
2. Interrogation: Too much per slide, centered everything, default colors
3. Reference: Apple (white space), Stripe (typography hierarchy)
4. Techniques: One idea per slide, asymmetric layouts, large imagery
5. Polish: Custom slide master, build animations, consistent positioning
6. Result: Impactful presentation that holds attention

### Pattern: Document Creation

1. Functional: Text in sections, default fonts
2. Interrogation: Poor hierarchy, no visual rhythm, cramped spacing
3. Reference: Swiss Design (grids), Notion (block structure)
4. Techniques: Serif headers + sans body, 12pt spacing system, generous margins
5. Polish: Page numbers, section breaks, high contrast
6. Result: Professional document that's easy to navigate

### Pattern: Data Table Elevation

1. Functional: Data in cells, standard formatting
2. Interrogation: Hard to scan, no visual hierarchy, default borders
3. Reference: Figma (properties panels), Swiss Design (precision)
4. Techniques: Bold headers with background, subtle banding, right-align numbers
5. Polish: Custom number formats, hide gridlines, highlight totals
6. Result: Readable table that communicates clearly

## Troubleshooting Common Elevation Failures

### Problem: "It looks better but still feels generic"
**Diagnosis:** Chose safe moves over bold ones
**Fix:** Push one element dramatically (3x larger headline, cut 50% of content, go asymmetric)

### Problem: "It's too busy now"
**Diagnosis:** Applied too many techniques at once
**Fix:** Remove decoration, focus on 1-2 core moves, increase white space

### Problem: "It doesn't match the context"
**Diagnosis:** Wrong exemplar or technique selection
**Fix:** Reassess context (corporate? creative? technical?) and choose appropriate references

### Problem: "Hierarchy is still unclear"
**Diagnosis:** Scale differences too subtle
**Fix:** Make primary elements 3-4x larger, make labels 0.75x smaller, increase contrast

### Problem: "Colors feel off"
**Diagnosis:** Too many colors or wrong palette for context
**Fix:** Reduce to 1 primary + 1 accent + grays, reference context-appropriate palette

### Problem: "Layout feels cramped"
**Diagnosis:** Not enough white space
**Fix:** Double all spacing, increase margins by 50%, remove elements if necessary

## The Essential Truth

**Elevation is not optional decoration—it's how design thinking gets applied.**

A functional output that hasn't been elevated is a first draft. A polished output that's been through this protocol is a finished piece.

The user shouldn't see the process. They should see only the result: visual work that looks like someone who knows what they're doing made it with care.

**Don't skip phases. Don't rush to done. The difference between generic and elevated is in the discipline of this process.**
