# Decision Framework

## Overview

Navigate difficult decisions using evidence-based frameworks that challenge stories with reality. Each framework surfaces blind spots, past patterns, and the gap between aspirational thinking and actual behavior.

**Core principle**: Most bad decisions stem from believing narratives that don't match reality. These frameworks force confrontation with data, past patterns, and honest self-assessment.

## Framework Selection Guide

Choose the framework based on your decision type:

**Priority & Life Decisions** → Year-to-Live Clarifier
- Major life choices, career decisions, feeling lost or overwhelmed
- Forces brutal prioritization through mortality salience
- Cuts through status/ego-driven thinking

**Self-Narrative Challenges** → Narrative Reality Check
- "I'm a morning person" / "I need to work 60+ hours" / "I'm productive when..."
- Reveals gaps between self-story and actual behavior patterns
- Pulls real data from calendar, email, Limitless to challenge beliefs

**Impulse Purchases** → 7-Day Cooling-Off Protocol
- New gadgets, upgrades, "limited time offers", subscription commitments
- Interrogates discount psychology and FOMO
- Analyzes past impulse purchase patterns
- Fast-track option for genuinely urgent decisions

**Time Commitments** → Time Allocation Audit
- Saying yes to new projects, quarterly planning, feeling overwhelmed
- Exposes fantasy time budget vs reality
- Forces specific trade-offs (not "I'll find time")
- Pulls data from calendar and activity logs

**Upgrade Decisions** → Incremental Value Calculator
- Considering "better" versions of things you already own
- Air fryer #4, new phone, upgraded software, premium subscriptions
- Calculates actual $ per new capability
- Includes downgrade test and satisfaction paradox analysis

**Ongoing Commitments** → Pre-Commitment Post-Mortem
- Before joining boards, taking advisor roles, starting projects, new relationships
- Predicts why you'll quit before you start
- Pattern recognition from past commitment failures
- Forces exit strategy planning

## Decision-Making Workflow

### Step 1: Identify the Framework

Determine which framework matches your decision type using the Selection Guide above.

### Step 2: Create Decision Document

Use the detailed template from `references/frameworks.md` for your chosen framework. Each template includes:
- Structured questions that surface blind spots
- Sections for gathering evidence and data
- Reality-testing checkpoints
- Future review triggers

**Create a new document** (Google Doc, Notion page, markdown file) and fill out the template systematically. Don't skip sections—they're designed to catch rationalizations.

### Step 3: Gather Evidence

Most frameworks require pulling actual data:

**For Narrative Reality Check & Time Allocation Audit:**
- Use `list_gcal_events` to pull calendar patterns from last 30-90 days
- Use `search_gmail_messages` to check email timestamp patterns
- Use `searchLifelogsWithTranscripts` (Limitless) to review actual behavior
- Check screen time reports, activity logs, commit histories

**For Cooling-Off Protocol & Incremental Value:**
- Research past purchase patterns (search Drive, email, receipts)
- Look up actual product features vs marketing claims
- Check historical sale patterns from the brand
- Find past similar purchases and audit their current usage

**For Pre-Commitment Post-Mortem:**
- Search Drive/Notion for past similar commitments
- Review calendar for actual time spent on previous commitments
- Search email for communication patterns from past obligations

### Step 4: Complete the Framework

Work through each section honestly. The frameworks are designed to:
- Challenge aspirational thinking with evidence
- Surface past patterns that predict future behavior
- Force specific commitments (not vague intentions)
- Set review triggers for accountability

**Critical**: Don't rush. The cooling-off period is part of the process. Urgency is often manufactured.

### Step 5: Make the Decision

Each framework concludes with a clear decision section. Answer:
- YES/NO/DEFER (be explicit)
- Why (one-sentence reason rooted in evidence)
- Review trigger (specific date for checking if you honored this)

**For YES decisions**: Document specific trade-offs, prevention strategies for known failure modes, and success metrics.

**For NO decisions**: Note what you learned about your patterns.

### Step 6: Set Review Triggers

Every framework includes review checkpoints. Set actual reminders:
- 7-day check (for cooling-off decisions)
- 30-day check (for most commitments)
- 90-day check (for narrative challenges)
- 6-month check (for major life decisions)

Reviews compare prediction vs reality and refine pattern recognition.

## Data Integration Points

These frameworks work best when fed with actual data, not memory:

**Google Calendar** → Time patterns, meeting loads, when work actually happens
- `list_gcal_events` with date ranges for last 30-90 days
- Analyze timestamps for productivity patterns
- Count actual vs estimated meeting hours

**Gmail** → Communication patterns, response times, email behavior
- `search_gmail_messages` with timestamp analysis
- Check when you're most responsive
- Review past purchase confirmation emails for pattern analysis

**Limitless** → Conversation transcripts, actual stated commitments
- `searchLifelogsWithTranscripts` to review what you actually said
- Find gaps between stated intentions and actions
- Review meeting transcripts for commitment patterns

**Google Drive** → Past decision documents, project histories
- `google_drive_search` for previous similar decisions
- Find past project timelines and outcomes
- Review old planning docs for reality vs prediction gaps

## Common Patterns and Insights

**Pattern: Discount-Driven Purchases**
- If you wouldn't buy at full price, you don't want the product
- "Limited time" offers usually repeat every 6-8 weeks
- Excitement for purchases typically lasts < 7 days

**Pattern: Aspirational Identity**
- "I'm a morning person" often means "I wish I was a morning person"
- Check: Last 30 days of calendar data usually tells the truth
- Gap: Calendar shows actual behavior vs identity story

**Pattern: Enthusiasm Decay**
- Initial excitement (8-10/10) typically drops to 4-6/10 by month 3
- Predict your quit point before committing
- Most people quit when enthusiasm hits 3-4/10

**Pattern: Time Optimism**
- Fantasy: "This will take 5 hours/week"
- Reality: Actually takes 8-12 hours/week
- Always add 50-100% buffer to time estimates

**Pattern: Incremental Upgrades**
- Upgrade satisfaction boost: Usually < 1 month before returning to baseline
- Cost per actual new capability: Usually higher than expected
- Most valuable test: "Would I buy this from zero?"

**Pattern: Commitment Failure Modes**
- Review past similar commitments for patterns
- Common quit reasons: Time creep, advice ignored, misaligned values
- If you can't articulate specific failure prevention, don't commit

|---------------|-----------|-------------|
| Life priorities, career | Year-to-Live Clarifier | Mortality-driven clarity |
| Self-beliefs about habits | Narrative Reality Check | Evidence vs story |

| Ongoing obligations | Pre-Commitment Post-Mortem | Predict your quit reasons |

## Meta-Framework: Learning from Decisions

After using these frameworks 5-10 times, patterns emerge about your decision-making:
- Which narratives you tell yourself repeatedly
- What triggers your impulse decisions
- How accurate your time estimates are
- What types of commitments you reliably quit
- Where your aspirational vs actual self diverges

**Track these patterns**. The frameworks become more powerful when you know your personal failure modes.

## Creating Decision Artifacts

All frameworks produce markdown decision documents. Save these:
- Create a "Decisions" folder in Drive or Notion
- File by date and framework type
- Review quarterly to identify meta-patterns
- Use past decisions to inform future ones

**Template**: Name decisions like "2025-10-18_CoolingOff_EspressoMachine.md" for easy retrieval and pattern analysis.
