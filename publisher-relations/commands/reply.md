# Publisher Reply

Draft responses to publisher communications with full context awareness.

## Execution

1. Identify which publisher and which email thread
2. Load publisher profile (relationship history, tone, formality level)
3. Check Notion for active project context
4. Check stored rates if discussing money
5. Draft reply:
   - Professional, warm, efficient tone
   - Sign as "Pavi"
   - Include clear action items and specific timelines
   - Reference relevant project context
6. **Present full draft in Biz Mode** — never send without explicit approval

## Tone Guidelines

- Publishers are professional relationships — warm but not casual
- Be solution-oriented when issues arise
- Protect professional boundaries while remaining friendly
- If negotiating, never concede on rate without Pavi's explicit approval
- For new project offers: express enthusiasm for the work, provide clear next steps

## Common Scenarios

### New Project Offer
"Thanks for thinking of me for [Title]. I'd love to take a look — could you send the manuscript? My current availability opens up [date], and I could target delivery by [date]. Happy to discuss further."

### Delivery Confirmation
"All files have been uploaded to [Box/FTP]. [N] chapters, [N] finished hours total. Invoice attached. Let me know if you need anything else."

### Schedule Request
"I can fit [Title] into my schedule starting [date]. Based on the word count, I'd estimate delivery around [date]. Does that work for your timeline?"

### Rate Discussion
[ALWAYS check stored rate first. If publisher proposes a different rate, present both to Pavi for decision. Never agree or counter without approval.]
